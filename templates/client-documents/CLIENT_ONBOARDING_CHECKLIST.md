# Client Onboarding Checklist

## 1. Client and Project Setup
- [ ] Confirm client legal name: `[Client Legal Name]`
- [ ] Confirm primary contact person and role
- [ ] Confirm project name and internal code
- [ ] Confirm project objectives and expected outcomes

## 2. Commercial Documentation
- [ ] Signed proposal received
- [ ] Signed service agreement received
- [ ] Quotation approved
- [ ] Initial invoice issued
- [ ] Initial payment confirmed

## 3. Compliance and Legal
- [ ] NDA signed (if required)
- [ ] Data processing expectations documented
- [ ] Regulatory requirements captured (`[Industry/Region Requirements]`)

## 4. Communication and Governance
- [ ] Kickoff meeting scheduled
- [ ] Communication channels agreed (`[Email/Slack/WhatsApp/Teams]`)
- [ ] Reporting cadence agreed (`[Weekly/Bi-weekly]`)
- [ ] Escalation contacts documented

## 5. Discovery Inputs
- [ ] Business goals and KPIs captured
- [ ] Existing systems and tools documented
- [ ] User personas or target customer segments shared
- [ ] Competitor/references shared

## 6. Access and Technical Readiness
- [ ] Domain/hosting access (if applicable)
- [ ] Repository access
- [ ] Cloud/platform credentials
- [ ] Analytics and tracking access
- [ ] Brand assets (logos, fonts, colors)

## 7. Delivery Planning
- [ ] Final scope baseline confirmed
- [ ] Milestone plan approved
- [ ] Dependencies and risks logged
- [ ] Acceptance criteria documented

## 8. Post-Launch Planning
- [ ] Support model selected (`[Essential/Professional/Enterprise/Custom]`)
- [ ] Warranty period expectations aligned
- [ ] Handover artifacts agreed
