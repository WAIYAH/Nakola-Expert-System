# Project Handover Checklist

## 1. Delivery Confirmation
- [ ] Final scope delivered and demonstrated
- [ ] Client acceptance documented
- [ ] Open issues list reviewed and agreed

## 2. Code and Infrastructure
- [ ] Source code repository transferred/shared
- [ ] Branching and release notes documented
- [ ] Environment details shared (dev/staging/production)
- [ ] Hosting/cloud access handed over

## 3. Credentials and Security
- [ ] Admin accounts transferred
- [ ] Shared passwords rotated
- [ ] MFA enabled where applicable
- [ ] Security checklist completed

## 4. Documentation Pack
- [ ] Technical architecture summary
- [ ] API documentation
- [ ] Deployment/runbook guide
- [ ] User manual or SOPs
- [ ] Known limitations and roadmap items

## 5. Quality and Performance
- [ ] QA summary report shared
- [ ] Test coverage summary shared
- [ ] Performance benchmark report shared
- [ ] Monitoring/alerting setup confirmed

## 6. Training and Enablement
- [ ] Admin training completed
- [ ] End-user training completed
- [ ] Recording/materials shared

## 7. Commercial and Support Transition
- [ ] Final invoice issued
- [ ] Payment reconciliation completed
- [ ] Warranty period start and end dates confirmed
- [ ] Support agreement activated (`[Plan Name]`)

## 8. Closure
- [ ] Project retrospective completed
- [ ] Reference/testimonial request sent (optional)
- [ ] Handover sign-off
